# Figma — Роадмап и учебный курс (урок + практика)

Этот роадмап спроектирован по образцу остальных разделов проекта: каждый блок — самостоятельный урок с целью, ключевыми темами, примерами и практическим заданием. Подойдёт как для новичка, так и для тех, кто хочет систематизировать знания по интерфейсному дизайну в Figma.

Рекомендация по форматированию курса: для каждого блока иметь два файла — `NN-topic.md` (теория + примеры) и `NN-topic-practice.md` (практика и задания). Для примеров хранить исходники в `handbook/frontend/design/figma/examples/`.

---

## Коротко: зачем учить Figma

- Figma — современный инструмент для UI/UX дизайна, совместной работы и прототипирования.
- Подходит для создания wireframes, UI kits, интерактивных прототипов и экспортов ассетов для фронтенда.

## Окружение и ресурсы

- Зарегистрируйтесь в https://figma.com (есть бесплатный план)
- Установите Desktop App (опционально) и подключите плагин Figma для VS Code (по желанию)
- Рекомендуемые ресурсы: Figma Learn (https://www.figma.com/learn/), YouTube (DesignCourse, Flux), статьи в Medium, UI Kits (Material, Ant Design)

---

## Структура курса — блоки

Каждый блок — 1 урок (~30–90 минут) + практика (1–3 задания). После 4–6 блоков — мини‑проект; в конце — финальный проект (полнофункциональная UI‑комната или SPA макет).

### Блок 01 — Введение в Figma (файл: `01-intro.md`)
Цель: освоить интерфейс Figma, документы, файл/фрейм, слои, экспорт.

Темы:
- Рабочее пространство: toolbar, layers, assets, inspector
- Фреймы и канвас, страницы, компоненты
- Инструменты: Move, Frame, Shape, Text, Pen
- Сохранение, экспорт PNG/SVG, версии файла

Практика:
- Создать файл, нарисовать простой макет страницы: header + hero + footer. Экспортировать логотип в SVG.

---

### Блок 02 — Основы композиции и сеток (файл: `02-layout-grid.md`)
Цель: понимать layout, constraints и автолэйаут.

Темы:
- Layout Grid: Columns/Rows/Margin
- Constraints: pinning/resizing поведения
- Auto Layout: вертикальные/горизонтальные стеки, padding, spacing

Практика:
- Создать карточку товара с использованием Auto Layout и Grid. Сделать адаптивный фрейм (mobile → desktop).

---

### Блок 03 — Типографика и цвета (файл: `03-typography-colors.md`)
Цель: настроить систему типографики и палитру, использовать Styles.

Темы:
- Text styles: H1/H2/body/caption
- Color styles: primary/secondary/neutral/accents
- Accessibility: контраст, размеры текста

Практика:
- Создать набор стилей (типографика + цвета) и применить их в простом лендинге.

---

### Блок 04 — Компоненты и дизайн‑система (файл: `04-components.md`)
Цель: понять, как строить переиспользуемые компоненты и их варианты.

Темы:
- Создание компонента, экземпляры, overrides
- Variants (кнопки: primary/secondary/disabled/hover)
- Component properties и автолэйаут внутри компонентов

Практика:
- Сделать библиотеку кнопок, input'ов и карточек; создать компонент с вариантами и пример использования.

---

### Блок 05 — Прототипирование и интерактивность (файл: `05-prototyping.md`)
Цель: создавать кликабельные прототипы и тестировать flow приложения.

Темы:
- Links, flows, transitions, smart animate
- Overlays, scrolling, interactive components
- Prototype preview and sharing (viewers, embed)

Практика:
- Сделать прототип для простого SPA (логин → список товаров → карточка товара) с базовой анимацией перехода.

---

### Блок 06 — Работа с сеткой и адаптивностью (файл: `06-responsive.md`)
Цель: проектировать интерфейсы, которые адаптируются под разные экраны.

Темы:
- Breakpoints: подходы к responsive design в Figma
- Auto Layout на уровне страниц, constraints и resizing
- Preview на разных размерах, экспорт макетов

Практика:
- Преобразовать desktop layout в mobile и tablet: переработать header и сетку контента.

---

### Блок 07 — SVG, иконки и экспорт ассетов (файл: `07-assets-and-icons.md`)
Цель: подготовить графику и оптимальные ассеты для фронтенда.

Темы:
- Векторные редакторы в Figma, экспорт SVG/PNG, оптимизация
- Создание иконок, объединение в components/asset library
- Настройка экспортных масштабов и форматов

Практика:
- Создать набор из 10 иконок, экспортировать в SVG и оптимизировать (svgo plugin рекомендации).

---

### Блок 08 — Работа в команде и дизайн‑хабы (файл: `08-collaboration.md`)
Цель: научиться совместной работе, комментариям и handoff разработчикам.

Темы:
- Комментарии, рецензирование, версияция
- Libraries and Team Libraries (паблиш компонентов)
- Inspect / Dev Mode: спецификации, CSS свойств и assets для разработчиков

Практика:
- Подготовить страницу для handoff: дать именованные слои, опубликовать библиотеку, создать guideline‑панель для разработчика.

---

### Блок 09 — Плагин‑экосистема и автоматизация (файл: `09-plugins.md`)
Цель: ускорять работу при помощи плагинов и делать рутинные задачи автоматическими.

Темы:
- Полезные плагины: Content Reel, Autoflow, Iconify, Measure, Figmotion
- Создание простого плагина (обзор) и макросы

Практика:
- Установить 5 плагинов, автоматизировать экспорт и генерацию placeholder content для карточек.

---

### Блок 10 — User Flows и UX‑методики (файл: `10-ux-flows.md`)
Цель: проектировать пользовательские сценарии и перевести их в экраны.

Темы:
- Customer journey maps, user flows, sitemap
- Wireframing → hi‑fi прототип
- Usability and cognitive load basics

Практика:
- Спроектировать user flow для функции «покупка» и сделать wireframe + hi‑fi прототип для 3 экранов.

---

### Блок 11 — Мини‑проект: посадочная страница (файл: `11-mini-project.md`)
Цель: собрать все знания в небольшой проект — responsive Landing Page.

Практика:
- Дизайн desktop + mobile, собрать компоненты в библиотеку, подготовить assets и handoff‑страницу.

---

### Блок 12 — Финальный проект (файл: `12-final-project.md`)
Цель: подготовить полноценный дизайн для SPA (dashboard, e‑commerce или SaaS) с библиотекой компонентов и прототипом.

Требования:
- Дизайн минимум 5 экранов (desktop + mobile)
- Компонентная библиотека с вариантами
- Интерактивный прототип (flows)
- Handoff: экспорт CSS, ассетов, спецификаций

Оценка:
- Читаемость и системность компонентов
- Адаптивность и качество прототипа
- Готовность к handoff разработчику

---

## Формат урока (шаблон)

Каждый урок должен содержать:
1. Введение — зачем это нужно
2. Краткая теория с примерами и скриншотами
3. Пошаговая часть: как это сделать в Figma (с пояснениями)
4. Практическое задание (1–3 задачи)
5. Критерии проверки и чеклист для self‑review
6. Ресурсы и примеры (UI kits, плагины, статьи, видео)
