04 — Lists and Tables

Правильное использование списков и таблиц. Семантика, доступность и адаптивность.

Списки

Когда использовать <ul>, <ol> и <dl>.

Ненумерованные списки (ul)

Используется для неупорядоченных коллекций (навигация, пункты без порядка).

  • Простой элемент
  • И ещё один
  • И вложенный список:
    • Вложенный пункт A
    • Вложенный пункт B

Нумерованные списки (ol)

Используется, когда порядок важен (шаги, инструкции).

  1. Подготовить окружение
  2. Установить зависимости
  3. Запустить проект

Описание (dl)

Список терминов и определений.

HTML
Язык разметки для веб-страниц.
CSS
Стиль и визуальное оформление страницы.

Стилизация списков для интерфейсов

Иногда нужен очищенный список без маркеров — используем list-style:none.

  • Быстрый доступ
  • История изменений
  • Настройки

Таблицы

Таблицы — для табличных данных, не для layout'а.

Доступность таблиц

  • Всегда используйте <th scope="col"> или scope="row" для заголовков
  • Добавляйте <caption> для описания содержания
  • Используйте aria-describedby или summary (legacy) для длинных таблиц

Комплексные таблицы

Для сложных таблиц используйте aria-labelledby, colgroup и правильные scope атрибуты.

Советы и частые ошибки

  • Не используйте таблицы для layout'а — используйте CSS Grid/Flexbox
  • Не забывайте о вложенных списках — сохраняйте читабельную иерархию
  • Тестируйте таблицы с клавиатурой и скринридерами
