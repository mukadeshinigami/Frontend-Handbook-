üìö Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the structure and content of the page, while CSS handles the styling and JavaScript adds interactivity.

Every HTML document follows a consistent structure that tells the browser how to interpret and display the content. This structure is essential for proper document rendering and accessibility.

üí° Did you know?

The first version of HTML was created by Tim Berners-Lee in 1991. Today's HTML5 is a living standard maintained by the WHATWG.

üèóÔ∏è Document Structure

A basic HTML document consists of several key elements:

  • <!DOCTYPE html> - Declares the document as HTML5
  • <html> - Root element containing all other elements
  • <head> - Contains metadata and configuration
  • <body> - Contains the visible page content

The structure ensures that browsers understand how to parse and render your document, and it also helps search engines understand your content better (SEO).

üè∑Ô∏è Common HTML Elements

Here are some of the most commonly used HTML elements:

  • <h1> - <h6> - Headings (h1 is most important)
  • <p> - Paragraphs of text
  • <a> - Hyperlinks
  • <img> - Images
  • <ul>, <ol>, <li> - Lists
  • <div>, <span> - Generic containers
  • <button> - Interactive buttons
  • <form>, <input> - Form elements

‚ö†Ô∏è Important

Always close your tags properly! HTML requires that most elements have an opening tag and a closing tag: <p>...</p>

üéØ Semantic HTML

Semantic HTML uses elements that clearly describe their meaning to both the browser and the developer. This improves accessibility and SEO.

Semantic elements:

  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Main content of the page
  • <article> - Independent, self-contained content
  • <section> - Thematic grouping of content
  • <aside> - Sidebar or supplementary content
  • <footer> - Footer information

Using semantic HTML makes your code more readable, maintainable, and accessible to assistive technologies and search engines.

‚úÖ Best Practices

  • Always include <!DOCTYPE html> at the beginning
  • Set character encoding to UTF-8: <meta charset="UTF-8">
  • Include viewport meta tag for mobile responsiveness
  • Use semantic elements instead of generic divs
  • Keep your HTML organized and properly indented
  • Use meaningful class and id names
  • Validate your HTML using W3C Validator
  • Avoid inline styles - use CSS instead
  • Keep the head clean and organized
  • Always close your tags properly
